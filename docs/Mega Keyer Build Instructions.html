Mega Keyer Build Instructions v1.1

 

 

The Mega Keyer is a board built to use as much as possible of the K3NG code. To achieve a reasonable size and cost, an embedded/mini Arduino mega 2560 is used. There are few options to extend the features of the board through available headers and pins.

 

The dimension of the board is 80mmx81mm it was intended to be 80mmx80mm but some gremlins decided that I would not be. This is mostly annoying and not a problem, but something to keep in mind while drilling and assembling the case components.

 

A basic case is made of two acrylic sheets 84mmx85mmx3mm and a 3D printed faade. There may be other, better option to build a case, but this will get you started. If you decide to not use a 3D printed faade, the screen can be fixed to the metal standoffs (with some insulation), or to some wooden/plastic square pegs at least 50mm high.

 

The following instructions are just to get you started, and not necessarily to be followed strictly. This is what I did to build some boards, and it worked for me.

 

You dont need special tools only a 3mm drill bit, and a metric ruler to measure and center the plate. To program the Arduino, you will need a computer and the free Arduino IDE.

 

Before to start:

1/ check the Arduino mega you got. Power it on using the usb micro cable and a computer or phone power supply. If there is no red LED blinking, you most likely got one without the Arduino boot loader. To add the boot loader, you will need another Arduino or an AVR programmer. If you have neither, I would recommend that you return your Arduino and make sure you get one with the bootloader. If you have the tools, adding a bootloader if tricky but not difficult.

 

2/ The Arduino is not programmed, and you will need to upload the K3NG software. Make sure that you are willing to do that or find a buddy who can.

 

3/ the Arduino is using a CH340 chipset and may not be recognized by your PC until you install some drivers and upgrade. To check, connect the Arduino with a usb micro cable to your computer. You should see the new serial port in the Arduino IDE. If not, you should look for the ch340 drivers for your OS.

 

4/ There are known issue listed at the end.

Step 1 : Plate drilling

 

You will need to use the PCB as a guide, so this has to be done before any soldering.

Again : the plates are not square (84mmx85mm) make sure that they are oriented correctly.

The PCB is 80mm wide by 81mm long / hole centers are 3mm from edge. When reporting to the plate, hole centers will be 5mm from edge of plate.

 

Figure 1 PCB dimensions (millimeters)

Make sure the clean the edges of the plates as there maybe paper residue that could impact alignment.

This is how I did my setup:

 

You can use a 2mm drill bit on the edge of the PCB to have proper spacing and repeat that on two adjacent side. This should center the PCB.

 

You can then use a drill with a 3mm bit to make the 4 holes. Make sure to have some backing material under the plate to avoid shattering of the acrylic.

 

Step 2: Arduino headers preparation

 

While the PCB has all the holes for the headers , we need only the two lengthwise headers (21x2 and 16x2). The one of the 3x2 can be used to extend the number of pins available, and install the boot loader if not there (more on that later).

 

You have first to prepare the headers by cutting the 40 pins female headers (single row or dual row) to 21 and 16 pins long. You only need a cutting plier, snipping a position after the number you need.

 

Then, you mate male and female and go to the next step.

Step 3 : Solder U7

 

Because this 10k bused network resistor is under the Arduino, and it is not recommended to plug/unplug the board because of the number of pins creating too much friction, this has to be soldered before installed the Arduino headers.

 

Make sure install the pin with the dot on the resistor network in the square pad.

Step 4 : Solder Arduino Headers.

 

The first step is to solder the 2x3 headers to add digital pins and ICSP to provide a way to install the Arduino boot loader if necessary. These headers are facing up to be available when the Arduino is mounted. Following is an example of the digital pin header orientation.

In order to make sure that the headers are square and will have the proper spacing when the Arduino is installed, mate the headers and the Arduino in place and solder corners before to solder all the pins (note that I did not install the 2x3 header mentioned above to take the pictures, but they should be there).

 

 

Now that the Arduino and pins are aligned you can solder pins at the corner (Arduino side and PCB bottom side), verify the alignment and then go ahead and carefully make the 148 solder joint without creating briges.

 

Step 5: Power connector

Make sure that the connector is sitting square to the edge of the board, and flat on the PCB. The holes are quite big and will need a lot of solder to fill. Make sure that contact is good.

 

Step 6: L1/L2 10 uH fixed inductors

These inductors look like resistors, make sure that you identify them correctly (brown black black).

 

Step 7: R1, R2, R9 220 Ohm resistors

Install the three 220 ohm resistors (red-red-brown)

 

Step 8: R6, R7 1K Ohm resistors

Install the two 1K Ohms resistors (brown-black-red)

 

Step 9: R3, R4, R5 470 Ohm resistors

Install the three 470 ohm resistors (yellow-violet-brown)

 

 

Step 10: R8 2.2K Ohm resistor

Install the 2.2 Kohm resistor (red-red-red)

 

 

Step 11: C1, C2 100pF capacitors

Install the 2 100pF capacitors. They are marked 101. There is no orientation

 

Step 12: C3/C4 1pF capacitor

Install the two 1 pF capacitors. They are marked 1 or 10 depending on the type. There is no orientation.

 

 

Step 13: C5 10uF electrolytic capacitor

Install the 10uF electrolytic capacitor. There is an orientation, and the white band on the capacitor has to be aligned with the flat white mark on the footprint.

 

 

 

Step 14: U2, U3 PC817 optocouplers

Install the two PC817 optocouplers by respecting the dot mark on the component and aligning with the dot on the footprint. The marking in the component may be very faint as shown below:

 

 

Step 15: U8 10k isolated network

The resistor network has an orientation. Make sure to align the dot with the square pad on the footprint.

 

Step 16: U5 Stacked LED

Make sure to align the front of the LED stack square to the edge of the PCB and flat on the board.

 

Step 17: Headers

Cut breakaway male headers and solder while making sure that they are straight (square to the PCB) and flat. Soldering one corner first help repositioning if necessary before final install.

Install a jumper on H4 to have audio.

 

Step 18: 3.5mm jacks.

Make sure to install the jacks square to the edge of the PCB and flat on the board.

 

Step 19: U4 Rotary Encoder

Make sure to install the encoder square to the edge of the PCB and flat on the board.

The insertion may be difficult as the side pins are bent and snap into the holes. This may require some force but be careful not to bend the pins.

 

Step 20: 2n2222 NPN transistor

Install the transistor with the flat part aligned with the flat on the footprint.

 

 

Step 21: 5v Piezo

There is no orientation for this component.

 

 

Step 22: LCD screen

 

The LCD screen has the pin oriented on the side to facilitate the connection to the board, use a flat nose plier to bend the back. Using female female jumper wires you can now easily connect it to the board (there is a typo on the PCB SCA should have been SDA on the pin header). Step 24: Buttons (optional).

 

Buttons are used for switching to command mode or to playback memories.

If you are mostly using the keyer in the winkey mode, this may not be relevant/useful. And the switch on the rotary encoder can be used as button 0.

Depending on which option you select, you may have to update the options before to upload to the Arduino.

Step 23: Power on and programing.

The Arduino can be powered either through the USB port (regular 5v USB micro phone charger, or PC), or with a 9v (optimal) power supply (center positive, 2.5mm plug).

 

Boot loader programming (only if necessary)

If when powering on the board the Arduino doesnt have a red LED flashing, it means that it came without a bootloader. Usually the Arduino boot loader is preinstalled, but sometimes it may not be. To install it, we can use the 2x3 header we installed in step 4.

 

The header pin out is (USB port facing up ICSP header at the bottom):

There is a good explanation there, so I will not reproduce here.

 

K3NG programming

 

After the bootloader is available, the Arduino will have to be programmed with the K3NG keyer code. There are many options available, and you can experiment on you own. I have created a clone of the repository preconfigured for this keyer and will update it with new upstream release and additional features. The repository code to use is:
https://github.com/w6ipa/k3ng_cw_keyer/tree/mega-keyer-v1.1

 

Under releases:

https://github.com/w6ipa/k3ng_cw_keyer/releases/

 

you can select which one you want to download (there is only v1.0 right now).

You then unzip the archive in your Arduino IDE sketchbook directory.

You need to copy the folder :

<sketchbook>/k3ng_cw_keyer/libraries/K3NG_PS2Keyboard

to

<sketchbook>/libraries

 

And then add the hd44780 library required by this project using the library manager (search for bill perry) :

 

Select the board type (Arduino Mega2560) and the port created when you plugged the keyer in the USB port.

Then, you are ready to upload the code, and if successful, you should be greeted with the K3NG Hi screen and tone.

 

 

Step 24: The case

The BOM is listing two options : either you go with just the plates and some headers or choose to print a faade. You can experiment in this area, with location of the screen/buttons to suit your need. Unfortunately, standard PCB size seem to be 100mm which means that to fit some standard case you may have to remote some connectors.

Known Issues

 

1/ there is a bug in the k3ng software that prevents switching from CLI to Winkeyer mode. Follow here : https://groups.io/g/radioartisan/message/10128

2/ powering through the barrel connector with 12v makes the lcd flicker when characters are displayed but works fine when powered through the usb.
